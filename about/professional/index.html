軟體工程

軟體工程是為了解決軟體危機而提出的概念，並主要分為兩個層面，分別是軟體開發技術以及軟體專案管理。這兩個層面都是我所熱衷的，但其中近年又以軟體專案管理層面為主要研究方向，並積極研究敏捷文化與方法。

敏捷方法

熟悉 Scrum 與 KanBan，並能夠分享相關主題以及帶領團隊執行。在 Sharelike 工作期間，漸進的導入 Scrum，並總共執行 16 次以兩週為一迭代的衝刺，中間主持過 13 次回顧、12 次檢視、11 次規劃會議。

能協助評估團隊當前狀態是否適合執行特定敏捷方法，並且漸進式的推廣敏捷文化到團隊中。不拘泥於既有的敏捷框架，而是專注在建立團隊文化以及專屬的敏捷流程。

軟體開發

從研讀碩士學程以來，就積極往降低程式設計師開發負擔的方向研究，這方向涉及的層面很廣，從善用設計模式（Design Pattern）、編寫測試（Testing）、需求規格實例化（Specification by Example）、持續整合（Continues Integration，簡稱 CI）、持續交付（Continuous Delivery）都是以來努力在涉獵的方向，並且希望能運用在未來的工作環境中。

程式設計

網頁後端

Ruby PHP JavaScript Ruby on Rails Laravel RifeWave.js MySQL PostgreSQL Firebase Cloud Firestore

目前主要開發語言是 Ruby，有透過 Ruby on Rails 開發的經驗，曾在 Sharelike 任職時協助開發基於 Ruby on Rails 和 Sinatra 的後端程式。也有透過 Ruby on Rails 從零開始開發出一個可用產品的能力，包括為指導教授任教課程開發的「九宮格審題系統」，以及為台灣體育大學研討會開發的「論文投稿與審核系統」。也會透過Ruby 編寫腳本程式去處理資料，包括解析 GitLab 和 Asana 的 API，搭配自動化整理出每次衝刺的客觀性資料給團隊參考。

另外次要開發語言是 PHP，在 EZTABLE 實習時，就能夠直接參與後端團隊對 PHP 系統的維護與開發。從大二就開始為學校圖書館維護 PHP 的網站，並能不透過既有框架從零到有編寫產品，協助圖書館將既有的流程資訊化。像是開發「業務統計系統」，讓傳統透過 Excel 檔案與 Email 去統計的業務資料，透過該系統更方便的輸入與統整，並輸出成報表；以及傳統紙本作業為主的尋書流程，也開發成「圖書館尋書系統」讓流程更佳簡便、有效率。除此之外，亦有使用主流框架 Laravel 的經驗，為學生會編寫了「學權申訴系統」、「海報欄位管理系統」。

網頁前端

JavaScript HTML5 CSS3 SCSS Vue.js BootStrap 4 Semantic UI Bulma jQuery

在網頁前端慣用的框架是 Vue.js，曾用 Vue 1.x 接下嘉義大學生輔組防自殺宣導網站的專案；到了 Sharelike 後，也維護過公司基於 Vue 2.x 的前端專案；也常會用 Vue.js 開發自己的 Side Project。

應用程式

C# .Net Framework jQuery

在就讀碩士期間所參與的軟體專案以 C# 構成，除了協助新增新功能外，也負責製作新的使用者介面，包括外觀和程式邏輯，並且將各模組從主專案分離，並透過依賴反轉原則和主程式和模組之間解偶，得將各模組另外交給其他成員實作、個別測試，降低開發門檻與維護成本。

DevOps

Docker GitLab CI

有能力自己編寫 Dockerfile，對於 Docker 的基礎概念也有一定的聊解，曾在 PHPConf 講過兩場推廣 Docker 的議程。自身也會使用 Docker 架設服務、建制環境等。在 Sharelike 時，也有透過 GitLab CI 為專案建置測試、建置、部署自動化的經驗。
